# Single-Page Developer Portfolio Website

This single-page responsive portfolio website was developed as part of the **Introduction to HTML, CSS, & JavaScript** course requirements within the **IBM Java Developer Professional Certificate** curriculum on Coursera. The application showcases semantic HTML5 document structures, customized CSS3 layout modules, and core asynchronous JavaScript DOM interactions to deliver standard frontend functionality and interactive form management mechanics.

## Technical Elements Demonstrated

- **Semantic HTML5 Architecture:** Implemented clean document outlines using distinct structural tags (`<header>`, `<section>`, `<nav>`, `<footer>`) to maximize accessibility and cross-browser parsing compatibility.
- **Dynamic DOM Manipulation:** Leveraged native JavaScript event listeners to intercept form submissions, parse user input parameters, and programmatically append new tracking elements into the structural DOM grid in real time without refreshing the viewport.
- **Combined Hover States & Responsive Design:** Configured cascading rulesets mapping exact class inheritance matrices (such as `.topmenu:hover`) to drive typography shifts and layout scaling patterns across desktop views.
- **Asynchronous Modal Validation Flows:** Integrated a conditional popup confirmation screen structure (`showPopup(true)`) triggered explicitly through internal data routing parameters once client verification criteria are successfully fulfilled.

## Application Directory Architecture

The deployment is modularly isolated into three main codebase layers:
- **`index.html`**: Defines the central document skeleton housing the profile section, project cards, vectorized image asset arrays, and recommendation form schemas.
- **`style.css`**: Manages the uniform corporate branding color scheme, typography scales, flexbox alignment rules, and explicit navigation state modifiers.
- **`script.js`**: Controls the interaction engines handling client submission routing, array parsing, and modal layer visibility toggles.

## Core Features & Class Architecture

To comply with automated testing frameworks, the site relies on an explicit class naming structure:
- **`.profile_name`**: Sets custom font specifications and light color accents for identity rendering.
- **`.topmenu`**: Governs combined bolding and underlining effects on navigation selections during mouse-hover windows.
- **`.skill`**: A specialized container mapping technical competencies paired with scalable `<img>` badge icons.
- **`.project-card`**: Standard grid layout boxes isolating functional repository descriptions and external links.
- **`.recommendation`**: Dynamically expandable panels displaying text quotes and author attributes.
- **`.iconbutton`**: Encapsulates a functional home navigation anchor referencing `#home` to instantly redirect users back to the page header space.
